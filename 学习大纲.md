# MindCrew 学习大纲（面向面试的复盘版）

> 写给一个目标是「在面试里把这个项目讲到能扛住拷打」的人，而不是写给一个想做开源贡献的人。
> 两者的差别：前者要的是**讲清楚为什么这么设计**，后者要的是**能不能从零敲出来**。
> 网上说的「复现核心代码」是后者的标准，对面试基本无用。

---

## 一、先回答你那三个问题

### Q1：什么算「核心代码」？
**不是**「能不看代码默写整个类」。
**是**：在白板上画出数据流图，指着每个框能说出——
1. 这一步要解决什么问题（没它会怎样）；
2. 关键参数怎么定的（TopK、阈值、TTL）；
3. 失败了怎么降级；
4. 替代方案是什么、为什么没用。

按这个标准，MindCrew 真正的"核心代码"就 **18 个类**（见下面第三节清单），其余的 Controller / Mapper / Entity 只需要知道在哪、是什么角色就行。

### Q2：学到什么程度叫学懂？
四个可验证的标准，缺一不可：
- **画图**：闭卷在白板画出「七步 RAG 流水线 + Agent 工具回路 + MCP 双轨」三张架构图；
- **指代码**：随便拎一个核心类，能说出它的入参、出参、3 个关键分支、1 个边界条件；
- **讲权衡**：每个核心类能答出「为什么这么做」+「替代方案是什么」+「为什么没选替代方案」；
- **讲故障**：能从「Embedding 模型超时」「Rerank 模型挂了」「LLM 返回 JSON 不合法」三个故障切入，把整条降级链路讲完。

### Q3：面试讲不了代码，怎么扛拷打？
面试官的拷打几乎都是这 5 类问题之一，跟「能不能默写代码」基本无关：

| 拷打类型 | 例子 | 你的弹药 |
|:--|:--|:--|
| **为什么需要 X**（why） | 为什么要 RRF 不直接拿向量结果？ | 单路召回的盲区 + 实际 case |
| **设计权衡**（trade-off） | 为什么选 Milvus 不选 PgVector？ | 数据规模、QPS、运维成本 |
| **边界情况**（edge） | 文档分片在表格中间断开怎么办？ | 切片策略 + 章节标题保留 |
| **故障降级**（failure） | Rerank 服务挂了怎么办？ | 关键词频率评分降级 |
| **量化结果**（metrics） | 加 Rerank 后准确率提升多少？ | 你自己测的对比数据（必须自己测） |

**面试不是代码复述比赛，是设计决策答辩。** 真正的拷打弹药是「故事卡 + 数据卡」，不是「类名 + 方法名」。

## 二、学习顺序（按依赖关系排，从外到内）

### 阶段 0 · 定位（半天）
**目标**：知道这个项目在简历上是怎么吹的，每条 STAR 对应到代码哪里。

1. 读 [MindCrew简历版.pdf](MindCrew/MindCrew资料合集/MindCrew简历版.pdf) — 把你要在简历上写的每一条**亮点摘出来**（4-6 条）；
2. 读 [MindCrew-项目介绍.md](MindCrew/MindCrew资料合集/MindCrew-项目介绍.md) — 重点看「核心架构 Agentic RAG」「七步检索链」「MCP 双轨」「生产级容错」四节；
3. 读 PRD.docx — 注意：PRD 里说的 **langgraph4j 是 Java 框架**（不是 Python 的 LangGraph），看一眼项目里 `crew/orchestrator/CrewOrchestrator.java` 就知道在哪用了；
4. 翻 [MindCrew面试题目.pdf](MindCrew/MindCrew资料合集/MindCrew面试题目.pdf) — **此刻只看题目不看答案**，把你答不上来的题列成清单，这就是你后面学习的考点。

**产出**：一个 `简历亮点清单.md`，每条亮点 → 对应代码包路径 → 对应面试题号。

### 阶段 1 · 跑通（1-2 天）
**目标**：让系统在你本机能问答，眼见为实。

跟着 [MindCrew-部署启动.docx](MindCrew/MindCrew资料合集/MindCrew-部署启动.docx) 走：
- `docker-compose up` 起 MySQL / Redis / Milvus / MinIO；
- 配 DashScope API Key（`.env`）；
- 后端 `mvn spring-boot:run`，前端 `npm run dev`；
- 上传 2-3 个 PDF / DOCX 到知识库；
- 问 5 个不同类型的问题：概念题、条款题（"第X条"）、时效题（"2026 最新"）、追问题、偏好题（"我对花生过敏"）。

**关键动作**：开 F12 看 SSE 流；进数据库看 `agent_trace`、`mcp_calls`、`reflection_log` 三张表存了什么——这三张表就是你面试时最硬的证据。

**产出**：5 张截图 + 1 段 SSE 抓包 + 3 张 trace 表截图。面试时讲到「答案可追溯」直接掏出来。

### 阶段 2 · 主线代码精读（5-7 天）
按数据流向读，**不要按字母序**。每天 3-4 个类，每个类按下面的「读类五问」走。

**Day 1 · 入口与 Agent 主循环**
- [MindCrewChatController.java](MindCrew/src/main/java/com/simon/MindCrew/controller/) — SSE 怎么发的、`SseEmitter` 还是 `Flux<ServerSentEvent>`；
- `MindCrewAgent.java` — 核心：`ChatClient.prompt().tools(...).stream()` 怎么调；
- `AgentToolContext.java` — ThreadLocal 为什么不传参（这是面试高频拷打点）；
- `AgentState.java` — 状态机有哪些字段、谁更新它。

**Day 2 · 多路召回**
- `VectorRetriever.java` — Embedding → Milvus search，注意 1024 维、COSINE、`nprobe` 参数；
- `BM25Retriever.java` — Lucene IndexWriter 在哪建索引、SmartCN 分词器、为什么不用 IK；
- `ChineseTextTokenizer.java` — 自定义分词的细节。

**Day 3 · 融合与精排**
- `RRFFusion.java` — `score = Σ 1/(k + rank)`，k=60 为什么不是 30 或 100；
- `CrossEncoderReranker.java` — DashScope gte-rerank 调用，降级为关键词频率评分；
- `QueryRewriter.java` — 短查询跳过、改写失败回退；
- `retrieval/ContextCompressor.java` — 50 字前缀去重、800 字截断、Token 上限控制。

**Day 4 · Prompt 与自检**
- `PromptAssembler.java` — `[1]` `[2]` 来源标注怎么塞进 Prompt，画像 / 长期记忆 / 6 条历史的拼接顺序；
- `SafetyGuard.java` — 紧急关键词 13 类、置信度 0.7 阈值的兜底；
- `SelfReflection.java` — 事实一致性 / 完整性 / 来源匹配三维评分，最多 2 轮纠错。

**Day 5 · MCP 工具与双轨**
- `mcp/DocSearchTool.java` / `KeywordSearchTool.java` / `WebSearchTool.java` / `MemoryTool.java` — `@Tool` 注解、参数 schema；
- `McpToolsConfig.java` — `ToolCallbackProvider` 怎么把同一组 Bean 暴露给内部 ChatClient 和外部 MCP；
- `QueryRouter.java` — Function Calling 失败时的降级路径（正则 + LLM 双判）。

**Day 6 · 多 Agent 编排（langgraph4j 部分）**
- `crew/orchestrator/CrewOrchestrator.java` — 状态图节点、边、条件路由；
- `crew/agents/PlannerAgent.java` → `ResearcherAgent.java` → `CriticAgent.java` → `WriterAgent.java` — 四个角色的输入输出契约；
- 对照 [langgraph4j GitHub](https://github.com/bsorrentino/langgraph4j) 看 `StateGraph` API。

**Day 7 · 知识入库与动态配置**
- `service/knowledge/*` — 文档解析（PDFBox / POI）、切片（段落 + 句子 + 滑窗）、双索引写入；
- `AiConfigHolder.java` — 数据库配置 → `@RefreshScope` 或自实现的热加载、LLM 模型热切换。

#### 读类五问（每个核心类都按这个走）
对着每个类问自己 5 个问题，写到一个 `读类笔记.md` 里：

1. **职责**：这个类一句话讲是干嘛的？输入是什么、输出是什么？
2. **触发**：谁调它？调它的上游有几个？
3. **关键分支**：方法里的 if/else 有几条主线？每条线在什么场景被走？
4. **降级**：它抛异常时，上游怎么处理？它自己内部有没有 try-catch 兜底？
5. **替代**：如果不用它，用什么替代？为什么没用替代？

5 个问题答完，这个类就「学懂」了。答不完就还没懂。

### 阶段 3 · 横切关注点（2 天）
这些不在某个类里，但面试官最爱问：

- **缓存策略**：`RagCacheService.java` + Redis Key 设计（什么粒度、TTL 多久、击穿怎么办）；
- **长期记忆**：`ExplicitMemoryExtractor` 抽取 → Redis 30 天 TTL → 召回时怎么注入 Prompt；
- **安全认证**：`security/` 包，JWT 怎么签发、过期策略、和 SSE 长连接怎么共存；
- **审计**：`common/audit/` 的 `@Audited` AOP 切面；
- **异常体系**：`common/exception/` 和 `common/result/` 的统一返回。

### 阶段 4 · 八股串联（3 天）
**这一步很多人跳，但这是面试时拷打的真正主战场。**
[资料合集](MindCrew/MindCrew资料合集/) 里有 6 份八股：

| 八股文 | 在项目里的钩子（必须能讲到） |
|:--|:--|
| 八股文-Redis.docx | 长期记忆 TTL、缓存击穿（热点问题）、Lettuce vs Jedis、是否用了 Redisson 锁 |
| 八股文-MQ.docx | 项目里**没用**MQ，但面试官会问「文档批量处理」「Embedding 异步」为什么不用 MQ |
| 八股文-数据库.docx | `agent_trace` / `mcp_calls` 表设计、为什么用 MyBatis Plus 不用 JPA、索引怎么建 |
| 八股文-网络.docx | SSE vs WebSocket（项目两个都用了）、HTTP/2 流式、长连接断线重连 |
| 八股文-java.docx | ThreadLocal（`AgentToolContext` 是高频拷打）、SpringBoot 自动装配、AOP（`@Audited`） |
| 八股文-agent.docx | ReAct、Function Calling、MCP 协议、langgraph4j 状态图、Self-Reflection |

读每篇八股时，**强迫自己在每个知识点旁边写一句**「在 MindCrew 里对应 XX 类的 XX 行」。能写出来的就是你的弹药，写不出来的就是漏洞。

### 阶段 5 · 自测与拷打（2-3 天）
看到这一步再翻 [MindCrew面试题目.pdf](MindCrew/MindCrew资料合集/MindCrew面试题目.pdf) 的答案。

闭卷做：把面试题分成三档——
- **必答（30 题）**：项目核心相关，必须能 90 秒内讲完；
- **追问（20 题）**：八股串项目的；
- **送命（10 题）**：你压根不会的，要么硬背要么准备「我没用过但我知道思路」的话术。

每题录音自答，回放找毛病。**不允许的回答模式**：「就是…然后…接着…」（流水账）、「这个嘛…」（卡壳）、「具体我记不清了」（致命）。

## 四、面试拷打弹药库（这才是核心交付物）

### 4.1 「故事卡」(STAR) — 准备 6 张
每张一页，包含 Situation / Task / Action / Result，**结果必须有数字**（哪怕是自己测出来的）。

| 故事卡 | 准备的细节 |
|:--|:--|
| **Card-1 · Agentic RAG 比传统 RAG 准在哪** | 同一份知识库、20 个问题，传统 vs Agentic 的准确率对比；至少 2 个具体 case |
| **Card-2 · RRF 融合的必要性** | 单 Vector / 单 BM25 / RRF 三组对比；术语类问题 BM25 赢，模糊问题 Vector 赢 |
| **Card-3 · Cross-Encoder Rerank 的收益** | 加 Rerank 前 vs 后的 Top3 命中率 |
| **Card-4 · 降级链路** | 一次「Rerank 服务超时」的演练：监控 → 降级 → 用户感知 → 后续修复 |
| **Card-5 · ThreadLocal 上下文设计** | 为什么不直接传参（LLM Function Calling 你控制不了入参）、清理时机、线程池下的坑 |
| **Card-6 · MCP 双轨** | 同一组 `@Tool` Bean 怎么既给内部 Agent 又给外部 Claude Desktop 用 |

### 4.2 「数字卡」 — 必须背熟的量化指标
拷打里只要出现「多少」「几」「多大」，没数字 = 没做过。

- 嵌入维度：1024；Milvus 相似度：COSINE
- RRF 的 k：60；融合后候选集大小
- Rerank Top-K：6；单切片截断：800 字
- 上下文 Token 上限：`maxTokens × 2` 字符
- 自检置信度阈值：0.7；自纠错最多 2 轮
- 长期记忆 TTL：30 天
- 紧急关键词类别：13 类
- 对话历史窗口：6 条
- 知识库类型支持：PDF / DOCX / MD / TXT

### 4.3 「拷打模拟」 — 自答 12 题
面试前最后一周，每天对镜子答 3-4 题，**录音回放**：

1. 你这个 Agentic RAG 跟普通 RAG 到底差在哪？能不能不用 Agent？
2. ChatClient Function Calling 失败你怎么知道？为什么要降级到 QueryRouter？
3. `AgentToolContext` 用 ThreadLocal，线程池场景下会不会污染？
4. Milvus 为什么选 COSINE 不选 L2？1024 维是怎么决定的？
5. 用 BM25 还要 Vector 干嘛？两路冲突时听谁的？
6. RRF 的 k=60 是怎么调出来的？换成 30 会怎样？
7. Rerank 服务挂了，降级用关键词频率，准确率掉多少？用户能感知吗？
8. SSE 流式输出过程中后端报错怎么办？前端怎么续传？
9. MCP 协议跟普通 HTTP API 比有什么本质区别？为什么不直接暴露 REST？
10. 长期记忆放 Redis 30 天，用户清除痕迹的需求怎么满足（GDPR）？
11. 多个 Agent（Planner/Researcher/Critic/Writer）为什么要用 langgraph4j 不用普通方法调用？
12. 自检循环最多 2 轮，为什么不是 3 轮、5 轮？

每题答完，对照下面这个评分表自评：

| 评分 | 标准 |
|:--|:--|
| A | 30 秒内点题 + 给出权衡 + 给出数字 + 给出替代方案 |
| B | 能讲清楚但缺数字 / 缺替代方案 |
| C | 流水账复述代码 |
| D | 卡壳 / 答非所问 |

C 以下的题就是漏洞，回去补阶段 2-3。

## 六、最常见的三个学习误区（别踩）

1. **逐字读完整个 src** — 90% 的 Controller/Mapper/Entity 是 CRUD 模板，读了忘、忘了读，浪费时间。盯着白名单 18 个类就够。
2. **不跑系统只读代码** — 面试官一句「你说说 SSE 实际返回的是什么格式」就破功。必须自己跑过、抓过包、看过 trace 表。
3. **背代码不背权衡** — 「这里用了 RRF」是无效信息，「为什么用 RRF 而不用加权平均」才是答案。每个技术选择都要准备一句「为什么不是别的」。

## 七、最后一句

你的对手不是这个项目，是面试官的「这是不是简历项目」雷达。
能扛住雷达的，从来不是把代码默写得多熟，而是**讲设计决策时眼里有光、嘴里有数字、心里有备选方案**。

按这份大纲走完，你不一定能从零写出 MindCrew，但你能讲明白它为什么是这个样子。
而面试要的就是后者。
